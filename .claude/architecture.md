# Architecture

## Layered Architecture

Dependencies flow **downward only**. A layer may import from any layer below it, never above.

```
┌─────────────────────────────┐
│           UI (React)        │  ← Presentation, user interaction
├─────────────────────────────┤
│           API (FastAPI)     │  ← HTTP endpoints, request validation
├─────────────────────────────┤
│         Service             │  ← Business logic, orchestration
├─────────────────────────────┤
│        Repository           │  ← Data access, persistence
├─────────────────────────────┤
│         Config              │  ← App configuration, env loading
├─────────────────────────────┤
│         Types               │  ← Pydantic models, shared types
└─────────────────────────────┘
```

### Layer Responsibilities

| Layer | Python Package | TypeScript Path | Owns |
|-------|---------------|-----------------|------|
| Types | `src/types/` | `src/types/` | Pydantic models, TypeScript interfaces, enums |
| Config | `src/config/` | `src/config/` | Env vars, feature flags, secrets refs |
| Repository | `src/repository/` | `src/repository/` | DB queries, file I/O, external API clients |
| Service | `src/service/` | `src/service/` | Business rules, orchestration |
| API | `src/api/` | — | FastAPI routers, middleware |
| UI | — | `src/ui/` | React components, pages |

### Dependency Rule Enforcement

To verify no upward imports exist:

```bash
# Check Python — should return no results
grep -rn "from src.api" src/service/ src/repository/ src/config/ src/types/
grep -rn "from src.service" src/repository/ src/config/ src/types/
grep -rn "from src.repository" src/config/ src/types/

# Check TypeScript — should return no results
grep -rn "from.*\/api\/" src/service/ src/repository/ src/config/ src/types/
grep -rn "from.*\/service\/" src/repository/ src/config/ src/types/
```

### Cross-Cutting Concerns

- **Logging**: Structured JSON via `structlog`. Every service logs entry/exit with request context.
- **Auth**: API-level only (JWT). Services are auth-unaware.
- **Telemetry**: OpenTelemetry spans per service call. Traces propagated via context.
- **Error handling**: Each layer wraps errors in typed error classes, never bare exceptions.
